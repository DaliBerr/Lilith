# 法术构筑 Token 系统设计文档

> 版本：v0.4  
> 用途：整理当前法术构筑系统的完整 Token 设计、类别边界、解释规则、状态槽、元素反应与实现优先级。  
> 备注：本文档基于当前构筑骨架整理。它不是最终平衡表，而是用于后续实现、数值配置、UI 表达和内容制作的设计基线。

---

## 1. 设计目标

当前法术构筑系统的核心目标是让玩家通过 Token 排列写出一个“小型战斗程序”。

每个 Token 应当同时满足三层价值：

| 层级 | 说明 | 示例 |
| --- | --- | --- |
| 单独可读 | 玩家看到字就能大致理解效果 | `火` 负责点燃，`冰` 负责冻结，`爆` 负责爆炸 |
| 位置有意义 | 同一个 Token 放在不同位置会被不同声明者解释 | `三` 可被 `散` 解释为散射数量，也可被 `爆` 解释为爆炸半径 |
| 可形成流派 | 多个 Token 能组成明确战术身份 | `雷 + 链 + 三` 是链电流；`火 + 触 + 聚 + 爆` 是集中爆破流 |

设计上应避免 Token 只是单纯数值词条。真正的乐趣应来自：  
**Token 类别、摆放顺序、作用域、Value 消费、Trigger / Payload 边界、以及元素状态反应。**

## 2. 当前语义基线

### 2.1 Token 类别

| 类别 | 职责 |
| --- | --- |
| `Core` | 生成可发射投射物的基础核心 |
| `Behavior` | 绑定附近 Core，改变投射物移动方式或飞行行为 |
| `Result` | 绑定附近 Core，或在 payload 中作为 result-only 效果 |
| `Value` | 被前方最近的声明者消费并解释 |
| `Modifier` | 按位置解析作用域，修饰 Core / Behavior / Result / CastBlock |
| `Multicast` | 创建同一次施法中的多个 projectile segment |
| `Trigger` | 绑定外层 projectile，触发后执行 payload |
| `Executor / Spellbook` | 法术书执行器，提供固定 Token、槽位、冷却、能量或原生修饰 |

### 2.2 基础解释规则

1. Token 从左到右解释。
2. `Core` 生成 projectile node。
3. `Behavior` 绑定附近 Core。
4. `Result` 绑定附近 Core，或在 Trigger 后的 payload 中作为 result-only 效果。
5. `Value` 不主动猜测意义，只能被前方最近的声明者消费。
6. `Modifier` 默认向后寻找合法目标。
7. `Modifier + Value` 中，Value 优先解释为 `NextN` 目标数量。
8. `Trigger` 后自动进入 payload。
9. payload 内 Modifier 只在 payload 局部向后解析。
10. 普通 Modifier 不自动变成全局修饰。
11. 当前默认不允许 nested Trigger / nested Payload。
12. 派生 projectile 默认不继承 payload，除非显式允许。

## 3. 命名与类别边界原则

### 3.1 避免同字跨类别

不建议让同一个字同时作为 Behavior 和 Modifier。  
例如：

- `疾` 保留为 Modifier。
- “飞行中逐渐加速”的 Behavior 改名为 `驰`。

推荐：

```text
疾 + 火
```

表示 `疾` 修饰后续 `火`，提高弹速。

```text
火 + 驰 + 三
```

表示 `火` 使用逐渐加速行为，`三` 解释为加速度。

### 3.2 Core 和 Result 的状态槽应统一

不建议出现：

```text
火 Core 积累一套点燃槽
燃 Result 积累另一套独立点燃槽
```

推荐统一为：

```text
点燃槽 = 一个状态槽
火 Core = 少量积累点燃槽
燃 Result = 大量积累点燃槽
火 + 燃 = 更快点燃
```

同理：

```text
毒 Core 与 蚀 Result 共用腐蚀槽
冰 Core 与冻结槽绑定
定 Result 与失能槽绑定
缚 Result 与绑缚槽绑定
```

### 3.3 派生效果必须防递归

以下 Token 都可能制造新 projectile 或实体：

```text
裂、分、召、傀、镜
```

必须默认限制：

1. 派生 projectile 不继承 Trigger / Payload。
2. 派生 projectile 不继承 `裂` / `分` 等继续派生能力。
3. 镜像触发不能再次触发镜像。
4. 召唤物不继承法术 payload。
5. 每次施法有 projectile / summon 上限。

## 4. Core Token

Core 是投射物基底。它应决定基础伤害、速度、寿命、碰撞方式、元素倾向和默认状态积累。

### 4.1 推荐 Core 清单

| Token | 类型 | 推荐定义 | 备注 |
| --- | --- | --- | --- |
| `箭` | 中性 Core | 默认装备。中速、中伤、无元素槽，兼容性最好 | 新手默认 Core |
| `火` | 元素 Core | 命中积累点燃槽，满槽后燃烧 DoT | 与 `燃` 共用点燃槽 |
| `冰` | 元素 Core | 命中积累冻结槽，满槽后冻结 | Boss 可转化为强减速 |
| `雷` | 元素 Core | 高速，命中后自带一次弱链电 | 与 `链` Behavior 叠加 |
| `岩` | 物理 Core | 低速高伤，贴地滚动，带击退/破甲倾向 | 对弹道行为要做兼容表 |
| `风` | 力场 Core | 高速低伤，命中点产生半径 3 的风压，水平推开低重量敌人 | 已实现；与 `斥` 区分：风不消费 Strength |
| `毒` | 异常 Core | 低速短射程，命中爆开毒 DoT，少量积累腐蚀槽 | 与 `蚀` 共用腐蚀槽 |
| `影` | 特殊 Core | 低伤，可穿越障碍物，命中敌人后正常结算 | 与 `弹` 对墙反弹冲突 |
| `刃` | 物理 Core | 快速、低伤、短射程，可多段切割 | 不要只是更快的箭 |
| `光` | 特殊穿透 Core | 高伤 projectile，可穿过敌人和墙体；每次穿透后后续直伤 *= 0.7 | 已实现为非 beam 特殊 projectile，只直伤，不触发 Result/Core 状态/OnHit/OnKill payload |
| `镜` | 功能 Core | 无直伤，命中后在镜像方向额外执行一次 Result / payload | 禁止镜像递归 |
| `召` | 功能 Core | 发射传送门，命中后释放召唤物 | 召唤物不继承 payload |
| `羊` | 特殊 Core | 命中积累变形槽，普通低重量敌人满 3 层后强控并变色 4 秒 | 已实现；不替换实体，高重量/Boss 不进入强控 |
| `谜` | 混沌 Core | 每发 projectile 发射时随机选择一个合法 Core 身份 | 已实现；候选池固定为 `箭/火/冰/雷/岩/刃/毒/影` |
| `水` | 元素 Core | 积累潮湿槽，低伤，参与水雷反应 | 已实现；不再用 `潮` 作为 Core 名，`潮` 保留 Result |

### 4.2 Core 内建倾向

| Core | 优势 | 弱点 |
| --- | --- | --- |
| `箭` | 泛用、稳定、低成本 | 无状态特色 |
| `火` | 持续伤害、爆发反应 | 对护甲一般 |
| `冰` | 控制强、安全 | 直伤低 |
| `雷` | 链接、清杂 | 单体不稳定 |
| `岩` | 高伤、破甲 | 慢，容易空 |
| `风` | 推开敌人、保命 | 爆发低 |
| `毒` | 持续和减抗 | 立即清怪弱 |
| `影` | 穿墙、绕障碍 | 伤害低 |
| `刃` | 高频、近距离压制 | 射程短 |
| `光` | 穿透、高威力 | 攻速低 |
| `镜` | 复制效果、几何玩法 | 无直伤，理解成本高 |
| `召` | 战场单位与牵制 | 实现和性能成本高 |
| `羊` | 趣味强控 | 需要精英/Boss 兼容规则 |
| `谜` | 高变化、高娱乐 | 平衡不可控 |

## 5. Behavior Token

Behavior 改变投射物的运动方式。当前建议每个 projectile 默认只保留一个主 Behavior。  
若未来要支持多个 Behavior，需要另行扩展组合规则。

### 5.1 推荐 Behavior 清单

| Token | 推荐定义 | Value 解释 | 备注 |
| --- | --- | --- | --- |
| `散` | 发射时产生多枚散射弹，每枚伤害衰减 | 散射数量 | 避免称为分裂，防止和 `裂` 混淆 |
| `穿` | 命中后继续飞行，伤害/速度逐次衰减 | 穿透次数 | `光` 可自带基础穿透 |
| `弹` | 撞墙或命中敌人后发生合理反弹 | 弹射次数 | 与 `影` 需要兼容规则 |
| `追` | 投射物追踪敌人 | 追踪强度 | 可被 `疾` / `久` 修饰 |
| `旋` | 投射物围绕施法者环绕 | 环绕半径 | 已实现；owner 丢失后沿最后有效锚点继续 |
| `链` | 命中敌人后跳转到其他敌人 | 链接目标数量 | 与 `雷` 自带弱链电叠加 |
| `滞` | 投射物停在发射点持续存在 | 持续时间 | 已实现；只做一次出生点直击检测，到期触发交给 `终` Trigger |
| `分` | 飞行期间均匀派生小弹 | 分裂次数 | 已实现；每次 2 枚半伤派生弹，不继承 payload 或继续派生 |
| `驰` | 飞行中逐渐加速 | 加速度强度 | 已实现；替代 Behavior 版 `疾` |
| `缓` | 飞行中逐渐减速 | 减速度强度 | 已实现；可与 `终` 形成延迟构筑 |
| `蛇` | 蛇形路径飞行 | 摆幅强度 | 已实现；固定频率，Value 控摆幅 |
| `游` | 确定性漂移路径 | 漂移强度 | 已实现；替代 Behavior 版 `乱` |

### 5.2 与 Modifier 的边界

以下不推荐作为 Behavior：

| 原想法 | 推荐归类 | 原因 |
| --- | --- | --- |
| `盾`：给飞行投射物添加小护盾 | 取消项：`卫` 仅文档留档 | 当前不再作为游戏内 Token 或 hidden prototype 计划 |
| `疾`：飞行逐渐加速 | Behavior `驰` 或 Modifier `疾` | 避免同字跨类别 |
| `乱`：随机路径 | Behavior `游` | `乱` 更适合作为高风险 Modifier |

## 6. Result Token

Result 表示命中后发生的效果。  
它可以绑定外层 Core，也可以位于 Trigger 后的 payload 内作为 result-only payload effect。

### 6.1 推荐 Result 清单

| Token | 推荐定义 | Value 解释 | 备注 |
| --- | --- | --- | --- |
| `爆` | 命中后爆炸 | 爆炸半径 | 基础范围伤害 |
| `裂` | 命中后分裂出小弹 | 分裂数量 | 小弹不继承 payload |
| `愈` | 命中后治疗玩家或友方 | 治疗范围半径 | 单体治疗可由半径 0 表示 |
| `定` | 积累失能槽，满槽后敌人不可攻击 | 爆条强度 | 可移动但不能攻击 |
| `燃` | 积累点燃槽 | 爆条强度 | 与 `火` 共用点燃槽 |
| `吸` | 命中后一次性拉近命中点附近低重量敌人 | 拉近强度 | 已实现；Strength 同时作为重量阈值和距离倍率 |
| `斥` | 命中后一次性推开命中点附近低重量敌人 | 推开强度 | 已实现；与 `风` Core 区分 |
| `缚` | 积累绑缚槽，满槽后敌人不可移动 | 爆条强度 | 敌人仍可攻击 |
| `蚀` | 积累腐蚀槽，满槽后减抗/破甲 | 爆条强度 | 与 `毒` 共用腐蚀槽 |
| `护` | 命中后给施法者添加临时吸收盾 | 护盾强度 | 已实现；`卫` 已取消，仅保留文档留档 |
| `幻` | 击杀目标后传送到目标位置 | 最大距离 | 更适合与 `灭` Trigger 配合 |
| `替` | 命中后与目标交换位置 | 最大距离 | Boss/精英需免疫或弱化 |
| `混` | 命中后随机触发一个合法 Result | 随机池等级或强度 | 应排除高风险 Result |
| `汲` | 命中后按直击伤害比例治疗施法者 | 吸血强度 | 已实现；替代原 `血` |
| `傀` | 击杀目标后复活为玩家傀儡 | 持续时间 | 替代原 `亡` |
| `标` | 给目标施加标记，后续伤害或追踪优先级提高 | 持续时间 | 强烈推荐补充 |
| `潮` | 积累潮湿槽 | 潮湿强度 | 支撑雷系反应 |
| `震` | 命中后打断、短硬直或冲击护甲 | 冲击强度 | 对迅捷/施法/护甲敌人有效 |
| `留` | 命中后留下持续伤害/状态区域 | 区域持续时间 | 已实现；半径默认 3，tick 默认 0.5 秒 |

### 6.2 Result 命名调整

| 原名 | 推荐名 | 理由 |
| --- | --- | --- |
| `血` | `汲` | 表达“汲取生命”，不像 Core 或资源 |
| `亡` | `傀` | 表达“复活为傀儡”，语义更具体 |

### 6.3 Result Value 规则

一个 Result 的 Value 最好只解释为一个主参数。

不推荐：

```text
吸 + 三
```

同时表示半径 3、强度 3、持续 3 秒。

推荐：

```text
吸 + 三
```

只表示吸引强度，半径和持续时间由默认值或 Modifier 决定。

## 7. Modifier Token

Modifier 不直接生成 projectile，而是修饰后续合法目标。  
普通 Modifier 按位置解析，不在资产上写死作用域。

### 7.1 通用 Modifier

| Token | 定位 | Core 映射 | Behavior 映射 | Result 映射 | 代价 |
| --- | --- | --- | --- | --- | --- |
| `疾` | 加速 | 速度提高 | 追踪响应/弹射速度提高 | 触发延迟降低 | 伤害略降 |
| `重` | 重击 | 伤害提高，速度降低 | 穿透/击退提高 | 爆炸伤害提高 | 冷却提高 |
| `锐` | 锐化 | 暴击/破甲提高 | 穿透保留率提高 | 对护甲增伤 | 范围降低 |
| `域` | 扩域 | 弹体尺寸提高 | 散射角提高 | 半径提高 | 单体伤害降低 |
| `久` | 延长 | 生命周期提高 | 追踪/滞留持续提高 | DoT/控制时间提高 | 速度降低 |
| `短` | 压缩 | 射程降低，伤害提高 | 更快触发 | 爆发倍率提高 | 安全性降低 |
| `稳` | 稳定 | 已实现：伤害 `*=0.9` | 已实现：角度扩散 `*=0.5` | 已实现：`蛇/游` 运动扰动 `*=0.5` | cast-level 特殊 Modifier，不消费 Value |
| `狂` | 狂暴 | 已实现：发射生命代价 `+5`，伤害 `*=1.35` | 已实现：本次能量消耗 `*=1.5` | 影响 projectile 直伤与派生 result 基准 | cast-level 特殊 Modifier，不消费 Value |
| `轻` | 轻盈 | 速度提高，击退降低 | 弹射次数提高 | 范围降低 | 伤害降低 |
| `贪` | 贪婪 | 已实现：发射生命代价 `+5` | 已实现：击败时所有掉落概率 `*=2` | 只消费本次死亡事件掉率倍率 | cast-level 特殊 Modifier，不消费 Value |
| `冷` | 冷化 | 冰/控制增强 | 减速增强 | 控制时间提高 | 火/燃烧弱化 |
| `烈` | 烈化 | 火/爆炸增强 | 散射更宽 | 爆炸/燃烧提高 | 自伤或过热 |
| `乱` | 随机修饰 | 随机实现一个合法 Modifier | 随机套用合法行为修饰 | 随机套用合法 Result 修饰 | 候选池过滤与可复现 |

### 7.2 Payload 专用 Modifier

| Token | 推荐定义 | 作用对象 | 注意 |
| --- | --- | --- | --- |
| `聚` | 缩小 payload 作用范围，提高密度/倍率 | payload 内 Result / inner projectile | 是 `域` 的反面 |
| `数` | 增加 payload 效果数量 | `裂`、`分`、多段 Result、inner multicast | `数 + 三` 仍解释为修饰后续 3 个目标 |
| `幅` | 增加 payload 效果倍率 | 爆炸伤害、治疗量、控制强度、吸引强度 | 强度来自资产，不来自后续 Value |

示例：

```text
火 + 触 + 聚 + 爆
```

命中后触发小范围高密度爆炸。

```text
火 + 触 + 数 + 裂
```

命中后增加分裂数量。

```text
火 + 触 + 幅 + 愈
```

命中后提高治疗倍率。

### 7.3 CastBlock Modifier

| Token | 推荐定义 | 生效范围 | 注意 |
| --- | --- | --- | --- |
| `放` | 修饰整个 CastBlock，扩大/放大本次施法 | 紧贴 Multicast 或法术书固定前置 | 与 `放 + 双 + 火 + 冰` 兼容 |
| `急` | 已实现：当前法术书施法间隔 `*=0.8` | 玩家 Spellbook executor | 继续遵守现有最小发射间隔；不消费 Value |
| `源` | 已实现：发射时法术书能量消耗 `*=0.5` | 玩家 Spellbook executor | 最终能量消耗不低于 0；不消费 Value |

### 7.4 Modifier + Value 规则

```text
疾 + 三 + 火 + 散 + 爆
```

解释为：

```text
疾 修饰后续 3 个合法目标：火、散、爆
```

不是：

```text
疾 的强度为 3
```

同理：

```text
数 + 三 + 爆 + 裂 + 愈
```

解释为：

```text
数 修饰 payload 内后续 3 个合法 Result
```

而不是 `数` 的强度为 3。

## 8. Value Token

Value 不单独生效。  
它必须被前方最近声明者消费，否则应给 warning。

### 8.1 数字 Value

| Token | 数值 | 用途 |
| --- | ---: | --- |
| `一` | 1 | 单体、精准、低成本 |
| `二` | 2 | 基础扩展 |
| `三` | 3 | 核心常用值 |
| `五` | 5 | 明显增强 |
| `八` | 8 | 高稀有、高风险 |

### 8.2 倍率 / 规模 Value

| Token | 推荐解释 | 注意 |
| --- | --- | --- |
| `半` | 0.5 倍 / 小规模 | 对 Count 类参数需决定取整规则 |
| `倍` | 2 倍 / 双倍规模 | 对 Count 可解释为数量 ×2，但 UI 必须写清 |
| `巨` | 大规模预设 | 不同参数类型映射不同 |
| `零` | 可选，高级 Token | 用于单体化、取消范围或特殊构筑 |

### 8.3 `巨` 的参数映射建议

| 参数类型 | `巨` 的解释 |
| --- | --- |
| Count | +2 或 ×1.5，封顶 |
| Radius | 大半径 |
| Duration | 长持续 |
| Strength | 高强度 |
| TriggerDistance | 长距离 |
| TriggerDelay | 长延迟，不一定是正收益 |

## 9. Multicast Token

Multicast 用于收集右侧多个 projectile segment。  
第一阶段建议写死收集数量，不急着让 Multicast 消费 Value。

| Token | 推荐定义 | 收集数量 | 发射方式 |
| --- | --- | ---: | --- |
| `双` | 双重施法 | 2 | 同时发射 |
| `叁` | 三重施法 | 3 | 同时发射 |
| `序` | 顺序施法 | 2 | 依次发射 |
| `绕` | 主从环绕施法 | 2 | 第一个为主弹，第二个围绕主弹 projectile |
| `叉` | 分叉施法 | 2 或 3 | 按角度分叉射出 |

### 9.1 后续扩展规则

若未来希望：

```text
序 + 三 + 火 + 冰 + 雷
```

表示三连顺序施法，需要正式新增：

```text
Multicast 可消费紧随其后的 Value 作为 requestedCount
```

否则第一阶段建议固定为：

```text
序 = count 2, pattern sequential
绕 = count 2, pattern orbit
叉 = count 2, pattern fork
```

## 10. Trigger Token

Trigger 绑定外层 projectile，触发后执行 payload。  
Trigger 后的内容默认进入 payload。

| Token | Trigger 类型 | 是否消费 Value | Value 解释 | 触发位置 |
| --- | --- | --- | --- | --- |
| `触` | OnHit | 否 | 无 | 命中点 |
| `时` | OnTimer | 是 | 延迟时间 | projectile 当前点 |
| `终` | OnExpire | 否 | 无 | 消失点 |
| `灭` | OnKill | 否 | 无 | 死亡目标位置 |
| `程` | OnDistance | 是 | 飞行距离 | projectile 当前点 |
| `近` | OnProximity | 是 | 感应半径 | projectile 当前点，另记录最近目标 |

### 10.1 Trigger 参数规则

> 如果 Trigger 声明 `ConsumesValueAsTriggerParameter = true`，则它会先消费紧随其后的 Value 作为 Trigger 参数，然后再进入 payload。

示例：

```text
火 + 时 + 三 + 爆
```

解释：火弹生成 3 秒后，在当前位置触发爆炸 payload。

```text
火 + 程 + 五 + 爆
```

解释：火弹飞行 5 单位距离后触发爆炸。

```text
火 + 近 + 三 + 定
```

解释：火弹靠近敌人 3 单位内时触发定身 payload。

### 10.2 Nested Trigger 限制

当前实现默认：

1. 默认不允许 nested Trigger。
2. nested Trigger 会产生 warning，内层不会开放继续触发。
3. `OnTimer`、`OnExpire`、`OnKill`、`OnDistance`、`OnProximity` 对同一个 payload 只触发一次。
4. 每次 payload 派生 projectile 有运行时预算上限。

不推荐第一阶段支持：

```text
火 + 触 + 火 + 触 + 冰
```

## 11. 状态槽系统

状态槽应统一管理，Core 和 Result 只是不同来源。

| 状态槽 | 主要来源 | 满槽效果 | 备注 |
| --- | --- | --- | --- |
| 点燃槽 | `火`、`燃`、火系反应 | 燃烧 DoT | `火` 少量，`燃` 大量 |
| 冻结槽 | `冰`、`冷` Modifier | 冻结，不能移动/攻击 | Boss 可转为强减速 |
| 潮湿槽 | `潮`、水系、冰融化 | 雷伤增强，火燃烧削弱 | 支撑元素反应 |
| 腐蚀槽 | `毒`、`蚀` | 减抗/破甲 | 不做两套腐蚀 |
| 失能槽 | `定` | 不能攻击 | 可移动 |
| 绑缚槽 | `缚` | 不能移动 | 可攻击 |
| 标记槽 | `标` | 后续伤害/追踪优先级提高 | 适合 Boss 战 |
| 变形槽 | `羊` | 普通低重量敌人满槽后强控 / 变色 | 高重量 / Boss 不进入强控 |
| 傀儡标记 | `傀` | 死后变为傀儡 | 与 `灭` Trigger 强相关 |

## 12. 元素反应

元素反应不应全部做成额外伤害。  
部分反应应改变战场、传播状态或增强控制。

| 反应 | 条件 | 效果 |
| --- | --- | --- |
| 热裂 | 点燃目标受到冰伤，或冻结目标受到火伤 | 清除部分冷热槽，造成爆发伤害 |
| 感电 | 潮湿目标受到雷伤 | 小范围链电，额外积累失能 |
| 导雷 | 冻结或绑缚目标受到雷伤 | 雷链目标数 +1 或链电范围提高 |
| 毒爆 | 中毒或腐蚀目标受到火伤 | 消耗毒层，产生毒火爆发 |
| 扩毒 | 毒云或腐蚀目标受到风伤 | 扩散毒云或传播腐蚀 |
| 震碎 | 腐蚀或护甲破裂目标受到岩伤 | 额外破甲伤害，可能短硬直 |
| 光蚀 | 被标记或腐蚀目标受到光伤 | 标记引爆，造成净化伤害 |
| 影噬 | 低血目标受到影伤 | 额外处决伤害 |
| 镜返 | `镜` 命中带状态目标 | 镜像方向复制状态爆发，但倍率降低 |

## 13. 示例构筑

### 13.1 基础构筑

```text
火 + 散 + 三
```

解释：火弹使用散射 Behavior，`三` 被 `散` 消费为散射数量。

```text
火 + 爆 + 三
```

解释：火弹命中后爆炸，`三` 被 `爆` 消费为爆炸半径。

```text
雷 + 链 + 三
```

解释：雷弹获得链接 Behavior，最多链接 3 个敌人。

### 13.2 Trigger / Payload 构筑

```text
火 + 触 + 爆 + 三
```

解释：火弹命中后，在命中点触发半径 3 的爆炸。

```text
冰 + 触 + 定 + 三
```

解释：冰弹命中后执行控制 payload，`三` 提高控制爆条强度或相关参数。

```text
毒 + 终 + 爆 + 三
```

解释：毒弹寿命结束后，在消失点触发半径 3 爆炸。

### 13.3 Modifier 构筑

```text
疾 + 火
```

解释：`疾` 修饰后续最近合法目标 `火`。

```text
疾 + 三 + 火 + 散 + 爆
```

解释：`疾` 修饰后续 3 个合法目标：`火`、`散`、`爆`。

```text
火 + 触 + 聚 + 爆
```

解释：payload 内 `聚` 修饰 `爆`，形成小范围高密度爆炸。

### 13.4 Multicast 构筑

```text
双 + 火 + 冰
```

解释：同次施法发射火和冰。

```text
放 + 双 + 火 + 冰
```

解释：`放` 紧贴 Multicast，作为 CastBlock Modifier 修饰整个双发结构。

```text
双 + 火 + 冰 + 触 + 爆
```

解释：第一段是普通火弹，第二段是命中后爆炸的冰弹。

### 13.5 高级构筑

```text
火 + 滞 + 三 + 终 + 爆
```

解释：火弹停滞 3 秒，寿命结束后爆炸。

```text
灭 + 傀
```

不推荐单独使用，因为缺少 Core。更推荐：

```text
火 + 灭 + 傀
```

解释：火弹击杀目标后，将目标复活为短暂傀儡。

```text
镜 + 触 + 爆
```

解释：镜弹命中后，在命中点与镜像方向额外执行爆炸；镜像触发不再递归。

## 14. Warning 与不推荐摆法

| 构筑 | 问题 | 建议 |
| --- | --- | --- |
| `三` | 没有 Core，也没有消费者 | 报 warning / error |
| `火 + 三` | Value 无人消费 | 改为 `火 + 爆 + 三` 或 `火 + 散 + 三` |
| `火 + 冰` | 重复 Core | 使用 `双 + 火 + 冰` |
| `火 + 散 + 追` | 重复 Behavior | 第一阶段只保留一个 Behavior |
| `火 + 爆 + 裂` | 重复 Result | 使用 payload 或未来扩展多 Result |
| `火 + 疾` | Modifier 在尾部，不回头修饰 | 改为 `疾 + 火` |
| `火 + 触` | Trigger 后没有 payload | 允许发射，但给 empty payload warning |
| `火 + 触 + 冰 + 雷` | Trigger 后全部进入 payload，`雷` 不回到外层 | 改为 `火 + 触 + 双 + 冰 + 雷` |
| `双 + 火 + 触 + 冰 + 雷` | 第一个 Trigger 吞掉后续 token，Multicast 不足 | 改为 `双 + 火 + 冰 + 触 + 雷` 或调整结构 |
| `火 + 触 + 火 + 触 + 冰` | nested Trigger 暂不支持 | 第一阶段禁止或限制深度 |

6. `召` 的召唤物是否可被 Modifier 影响？
7. `傀` 的傀儡生命值来自敌人原生命，还是固定表？
8. `幻 / 替` 对 Boss、精英、不可移动敌人的处理规则？
9. 元素反应除默认 50% 消耗外，哪些反应需要配置化覆盖？
10. 派生 projectile 是否需要继承 Core 以外的极少数安全字段？
11. 每次施法的 projectile / summon / payload 执行上限是多少？

## 18. 总结

当前推荐的正式 Token 清单如下：

```text
Core:
箭、火、冰、雷、岩、风、毒、影、刃、光、镜、召、羊、谜、水/潮

Behavior:
散、穿、弹、追、旋、链、滞、分、驰、缓、蛇、游

Result:
爆、裂、愈、定、燃、缚、蚀、标、潮、震、汲、护、留、斥、吸、幻、替、混、傀

Modifier:
疾、重、锐、域、久、短、稳、狂、轻、贪、冷、烈、乱
Payload 专用：聚、数、幅
CastBlock 专用：放
Spellbook / Executor 专用：急、源

Value:
一、二、三、五、八、半、倍、巨、零

Multicast:
双、叁、序、绕、叉

Trigger:
触、时、终、灭、程、近
```

核心设计结论：

1. `疾` 保留为 Modifier，逐渐加速 Behavior 改为 `驰`。
2. `盾` 不作为 Behavior；原 Modifier `卫` 已取消并从游戏侧移除，只在文档中留档；`护` 保留为给玩家护盾的 Result。
3. `火/燃`、`毒/蚀` 使用统一状态槽。
4. `雷` 的自带弱链电和 `链` Behavior 分层设计。
5. `镜` 可以作为功能 Core，但必须禁止递归。
6. `裂/分/召/傀` 等派生能力默认不继承 payload。
7. `数/幅/聚` 的强度来自资产；后续 Value 仍解释为 NextN 作用域。
8. `时/程/近` 使用 Trigger 参数消费规则；`终/灭` 当前无参数。
9. `序/绕/叉` 当前固定收集数量，不消费 Value。
10. UI 必须显示 Value 消费、Modifier 作用域、Trigger payload 和 Multicast segment。
